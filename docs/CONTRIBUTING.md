# Contributing to OpenTI

Thank you for your interest in contributing to OpenTI!  

## How to Contribute

1. Fork the repository  
2. Create a feature branch: `git checkout -b feature/your-feature`  
3. Make your changes  
4. Test your code thoroughly  
5. Submit a pull request with a clear description

---

## Code Style

- Python scripts: PEP8 compliant  
- Markdown documentation: Use proper headings, code blocks, and formatting  
- Commit messages: Present tense, concise, and descriptive

---

## Reporting Issues

Use the GitHub Issues tab to report bugs or request features. Include as much context as possible.

---

## Pull Request Guidelines

- Ensure tests pass before submitting  
- Update documentation if adding new functionality  
- Reference the issue number in your pull request
